JD_logs / Hide Player Details
  • View on github

Using the export with no player details:

PLAYER_ID: isn't used and therefore will be 0
PLAYER_2_ID: isn't used and therefore will be 0
exports.JD_logs:discord("EMBED_MESSAGE", PLAYER_ID, PLAYER_2_ID, 'DECIMAL_OR_HEX_COLOR_CODE', 'WEBHOOK_CHANNEL')
Example: (/tweet [Message]) 
RegisterCommand("tweet", function(source, args, rawCommand)
    TriggerClientEvent('chatMessage', -1, "Tweet | " .. GetPlayerName(source)..": "..rawCommand:gsub("tweet ", ""), { 201, 201, 201 })
    exports.JD_logs:discord("Tweet | " .. GetPlayerName(source)..": "..rawCommand:gsub("tweet ", ""), 0, 0, '#F1F1F1', 'tweet')
end)
EMBED_MESSAGE: This will be the /tweet message in this case
PLAYER_ID: Since the player info is not used this will be 0
PLAYER_2_ID: Since the player info is not used this will be 0
COLOR: This can be any color you want
CHANNEL: This will be linked to the webhook in the config.
