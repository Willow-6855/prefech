Using the export for one player:

PLAYER_ID: is the variable that is used for getting the player info.
PLAYER_2_ID: isn't used and therefore will be 0
exports.JD_logs:discord("EMBED_MESSAGE", PLAYER_ID, PLAYER_2_ID, 'DECIMAL_OR_HEX_COLOR_CODE', 'WEBHOOK_CHANNEL')
for server-side resources this will be source on client-side this will be GetPlayerServerId(PlayerId())
Keep in mind these might also change depending on the framework
Example: (/me [Message])
RegisterCommand("me", function(source, args, rawCommand)
    TriggerClientEvent('chatMessage', -1, "ME | " .. GetPlayerName(source)..": "..rawCommand:gsub("me", ""), { 201, 201, 201 })
    exports.JD_logs:discord("ME | " .. GetPlayerName(source)..": "..rawCommand:gsub("me", ""), source, 0, '#F1F1F1', 'me')
end)
EMBED_MESSAGE: This will be the /me message in this case
PLAYER_ID: In this use it will be source
PLAYER_2_ID: Since there is one player this will be 0
COLOR: This can be any color you want
CHANNEL: This will be linked to the webhook in the config.
